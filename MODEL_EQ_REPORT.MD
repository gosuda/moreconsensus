# Model Correspondence and Scope Report

This report maps the current finite TLA+ models to the Go implementation, compares the current model suite against the attached original `EgalitarianPaxos` TLA+ module, and names implementation areas verified outside the models.

## Model suite scope

The repository now uses a split model suite rather than one monolithic implementation oracle:

| Model | Scope |
| --- | --- |
| `tla/EPaxos.tla` | Normal-case pre-accept, accept, commit, local conflict-derived attributes, quorum-gated fast/slow transitions, and deterministic minimal-component execution. |
| `tla/EPaxosResponses.tla` | Response quorum evidence for fast commit, slow accept, prepare outcomes, prepare-driven commit/accept transitions, and bounded prepare-recovery branch priority including `TryWitnessQuorum = FastQuorum + SlowQuorum - ReplicaCount`. |
| `tla/EPaxosOptimizedRecovery.tla` | Focused finite optimized-recovery model: prepare fast-witness threshold, recovery-only AcceptReply evidence that does not mutate chosen attributes, and an Accept-Deps stale-dependency slice where chosen `Deps` already order the conflict. The gate runs 3-, 5-, and 7-replica configurations. It abstracts the read-only message exchange into evidence availability; `tla/EPaxosEvidenceQuery.tla` covers the message path separately. |
| `tla/EPaxosEvidenceQuery.tla` | Focused finite committed-conflict TryPreAccept evidence-query model: guard-gated `MsgEvidence` start, read-only evidence responses, duplicate/mismatched response drops, sender-preserving `AcceptEvidence` validation, stale TryPreAccept rejection restart, and fail-closed slow-accept fallback. The gate runs 3-, 5-, and 7-replica configurations. |
| `tla/EPaxosRecovery.tla` | Focused stopped-owner dependency recovery: a non-owner coordinator recovers a missing dependency without owner votes, commits a no-op, and releases the dependent command under fairness. The gate runs 3- and 5-replica configurations. |
| `tla/EPaxosRollbackAllocation.tla` | Focused finite rollback-allocation model: a restored local checkpoint learns a later own committed instance from quorum state, advances `nextInstance`, exercises the defensive allocation skip over an already-known future local ref, allocates a fresh local instance, and checks that the learned commit appears before the fresh post-catch-up allocation in the apply sequence. It does not model full EPaxos recovery, message loss, storage checksums, application commands, or unbounded rollback histories. |
| `tla/EPaxosConfigBarrier.tla` | Focused finite configuration-barrier model: two fixed configuration-command refs plus one user command check local `pendingConf` blocking, dependency-vector inclusion, sequence elevation, and pending clearing after all known config refs execute. It does not model dynamic membership transitions, quorum changes, recovery under configuration changes, or unbounded instance spaces. |
| `tla/EPaxosConfigTransition.tla` | Focused finite dynamic configuration-transition model: one old three-voter configuration, one add-voter successor configuration, a config command chosen/executed under the old configuration, one old in-flight user instance that remains pinned to the old voters/quorum after the transition, and one later user instance pinned to the new voters/quorum. It does not model remove-voter transitions, multi-step reconfiguration chains, joint consensus, recovery under configuration changes, durable replay, or unbounded configuration histories. |
| `tla/EPaxosRevisited.tla` | Focused EPaxos Revisited-inspired implementation checks: explicit TOQ zero-sequence/empty-dependency envelopes, delayed originator assignment at `ProcessAt`, pending-TOQ decision blocking, deterministic receiver `ProcessAt` processing/fast-wait timing, and chain-pruning execution. It abstracts real physical clock synchronization and OWD measurement into a finite timestamp. |
| `tla/TOQClockDiscipline.tla` | Finite TOQ embedder clock-discipline contract: if receiver clocks are within configured skew and one-way delays stay within configured bounds, the sender's `ProcessAt = send-local-time + MaxDelay + SkewBound` is late enough for every sync-group member to have received the flagged PreAccept by its local `ProcessAt`. In Go/API terms, each configured `TOQOneWayDelay[id]` value must already include the receiver's clock-skew/synchronization uncertainty. This model does not implement or prove clock synchronization, one-way-delay measurement, or sync-group construction. |
| `tla/ReadyAdvance.tla` | Durable `Ready` / `Advance` prefix acknowledgement and retry obligations. |
| `tla/Quorum.tla` | Supported quorum tables and quorum intersections. |
| `tla/KVTimestampStaleness.tla`, `tla/KVOmissionRecovery.tla` | Example KV timestamp semantics and omission/recovery scenarios. |


## Paper-grounded claim map

| Claim area | Paper claim | Current repository claim | Status |
| --- | --- | --- | --- |
| EPaxos fast quorum | EPaxos SOSP 2013 §4.3/§4.4, ACM proceedings pp.362-365, and CMU-PDL-13-111 §6 define the optimized fast quorum as `F + floor((F + 1) / 2)` including the leader for odd `N=2F+1`. This yields `N=3 -> 2`, `N=5 -> 3`, `N=7 -> 5`, and relies on matching recovery evidence. | `epaxos/quorum.go`, `tla/Quorum.tla`, `tla/EPaxos.tla`, and `tla/EPaxosResponses.tla` use the optimized paper threshold for odd supported sizes and keep the previous conservative threshold for even supported sizes (`N=4 -> 4`, `N=6 -> 5`) because the paper proof assumes odd `N`. Fast commits require matching-quorum FP-deps-committed prefix evidence in Go and the normal-case TLA model. | Implemented normal fast-path threshold/evidence for odd supported sizes; normal un-timed originator-disagrees fast commit, even-size optimized quorum proof, and unbounded proof remain documented non-claims. |
| EPaxos recovery | EPaxos SOSP 2013 §4.6, ACM proceedings p.366, and the technical report require prepare recovery, try-pre-accept conflict checks, and dependency evidence sufficient for the chosen fast quorum. | Go implements prepare priority over committed, highest-durable-record-ballot accepted, preaccepted try-candidates, preaccepted slow-accept fallback, and no-op branches; durable `FastPathEligible`; durable `InstanceRecord.RecordBallot` separate from promise `Ballot`; FP-deps-committed prefix evidence; TryPreAccept conflict checks including committed stale-dependency evidence-query/resend-ignore, targeted leader-in-fast-quorum/deferred-cycle Go slices, exact current-ballot `MsgAcceptResp` counting, and stopped-owner recovery against the implemented quorum table. `tla/EPaxosResponses.tla` checks bounded prepare branch priority, `TryWitnessQuorum`, and ballot-bound AcceptOK evidence; `tla/EPaxosRecovery.tla` checks stopped-owner dependency recovery; `tla/EPaxosEvidenceQuery.tla` checks finite read-only committed-conflict evidence-query guards, sender-preserving evidence validation, duplicate/mismatched drop behavior, stale-ballot restart, and fail-closed fallback. | Implemented Go recovery behavior for supported configurations; unbounded proof, complete optimized-recovery branch parity, and arbitrary membership-change recovery remain non-claims. |
| EPaxos Revisited TOQ | NSDI 2021 §4/§4.1, USENIX proceedings pp.616-617, requires synchronized physical clocks, one-way-delay estimates with clock-skew/sync margin and sync groups, outbound `PreAccept` messages with sequence number `0` and empty dependencies, and delayed proposer dependency assignment. | Go exposes explicit `Config.TOQ` separate from `TimeOptimization`; `epaxos/node.go` computes `ProcessAt` from `TOQClock` plus configured conservative delay bounds where `TOQOneWayDelay` includes one-way network delay plus clock-skew/sync uncertainty margin, persists `TOQPending`/`ProcessAt`, sends flagged zero-attribute TOQ PreAccepts, delays local assignment until `ProcessAt`, blocks decision while pending, and has receivers compute attrs at the flagged timestamp. The core does not synchronize clocks or measure one-way delay. | Implemented core TOQ behavior with caller-supplied synchronized-clock and OWD/skew-margin discipline; real clock synchronization/measurement remains outside the finite model claim. |
| EPaxos Accept-Deps recovery evidence | CMU-PDL-13-111 §6/§6.2 describes Accept-Deps evidence carried on Accept/AcceptReply for optimized-recovery corner cases. | Go stores recovery-only `AcceptSeq`/`AcceptDeps` plus sender-preserving `AcceptEvidence`, carries it on `MsgAccept`, `MsgAcceptResp`, `MsgCommit`, `MsgPrepareResp`, and read-only `MsgEvidenceResp`, ignores non-current-ballot `MsgAcceptResp` before merging evidence, and uses sender evidence for committed stale-dependency evidence-search/resend-ignore when chosen `Deps` already include the conflict. Legacy aggregate evidence cannot authorize that ignore; fail-closed cases slow-accept with the committed conflict sequence when known. `tla/EPaxosOptimizedRecovery.tla` checks finite 3-, 5-, and 7-replica evidence-use slices; `tla/EPaxosEvidenceQuery.tla` checks the finite read-only `MsgEvidence`/`MsgEvidenceResp` path and fail-closed sender-preserving evidence guards. | Implemented Go committed stale-dependency evidence-query/resend-ignore path for supported F<=3 configurations; unbounded proof remains a non-claim. |
| EPaxos Revisited chain pruning | NSDI 2021 §3, USENIX proceedings p.616, permits pruning a dependency already ordered through a higher-sequence transitive dependency. | Go and `tla/EPaxosRevisited.tla` model/check the implemented pruning rule. | Implemented claim. |

References: [EPaxos SOSP 2013](https://sigops.org/s/conferences/sosp/2013/papers/p358-moraru.pdf), [EPaxos technical report CMU-PDL-13-111](https://www.pdl.cmu.edu/PDL-FTP/associated/CMU-PDL-13-111.pdf), [EPaxos Revisited NSDI 2021](https://www.usenix.org/system/files/nsdi21-tollman.pdf).

## Production completion gate

This gate defines the minimum evidence required before the project can be called production-complete under a mission-critical, high-assurance bar. The gate is intentionally blocker-driven: any failed blocker row keeps the release closed.

## Open correspondence risks

- The TLA+ model is finite-bounded for TLC, so it demonstrates correspondence for the configured state space rather than unbounded proof.
- `tla/EPaxos.tla` is a normal-case integrated safety model; prepare branch priority and TryPreAccept witness thresholds are checked through `tla/EPaxosResponses.tla`, and stopped-owner dependency recovery is checked through the focused `tla/EPaxosRecovery.tla` model rather than inlined into the normal-case model.
- `tla/EPaxosConfigBarrier.tla` is a finite local bookkeeping model for two fixed config refs and one user command. It checks `pendingConf`, dependency, sequence, and clearing obligations only; dynamic membership transitions, quorum reconfiguration, recovery under config changes, durable storage, and unbounded config histories remain outside this model.
- `tla/EPaxosRollbackAllocation.tla` is a finite local rollback-allocation bookkeeping model. It checks a single older checkpoint, one own committed instance learned from quorum, one defensive known-future local ref, fresh allocation after skip, and learned-before-fresh apply sequence only; it does not model full EPaxos recovery, storage checksums, message loss, application commands, arbitrary multi-replica rollback, or unbounded rollback histories.
- Optimized EPaxos fast quorum is implemented for odd supported cluster sizes with FP-deps-committed prefix evidence, durable RecordBallot recovery evidence is implemented in Go, and recovery-only Accept-Deps evidence is implemented for Accept/AcceptReply/Commit/PrepareReply plus sender-preserving read-only evidence queries. Go also implements targeted leader-in-fast-quorum/deferred-cycle TryPreAccept slices, committed-conflict slow-accept hardening, and committed stale-dependency evidence-search/resend-ignore for supported F<=3 configurations. `tla/EPaxosEvidenceQuery.tla` now covers the finite read-only evidence-query path, but even cluster sizes keep the conservative quorum, and complete optimized-recovery branch parity plus unbounded proof remain outside the current finite model/release claim.
- `tla/EPaxosRevisited.tla` verifies finite explicit TOQ envelope, delayed assignment, pending-decision blocking, receiver `ProcessAt` processing, fast-wait, and chain-pruning obligations. It abstracts synchronized physical clocks, one-way-delay measurement, and sync-group construction into configured finite timestamps. `tla/TOQClockDiscipline.tla` separately checks a finite bounded-skew/bounded-delay contract that makes `ProcessAt` late enough for sync-group delivery before local processing; it still does not prove an operational clock-sync or delay-measurement implementation.
- Membership-change validation, invalid Ready-content validation, logical-timer dispatch, checksum, wire-decoder tolerance, storage virtualization, transport fault control, storage fault control, destructive-storage control, checkpoint semantic validation, live-source checkpoint recovery, optional external Jepsen lifecycle tooling, and example KV apply/scan storage semantics are verified by implementation and Jepsen tests but outside the current TLA safety state.
- Local Jepsen now covers loopback process restart with health-checked rejoin, transport partition, storage-unavailable, and destructive-storage removal that keeps a node down until original storage is restored. External multi-host deployment tooling exists for separately approved environments, but this simulation/local-loopback evidence bundle does not exercise those histories.
